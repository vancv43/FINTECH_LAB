Tổng quan Capstone — Lab B → Production
Slide 15–16 · Key Takeaway: "The Pattern Transfers Across Domains"
🚀
Mục tiêu cốt lõi (Slide 15): Nâng cấp Agentic Pipeline từ lab prototype lên hệ thống AI production-grade thực sự — với semantic search, explainable AI, multi-step reasoning, và một API có thể tích hợp vào enterprise workflow. Bạn không làm lại từ đầu — bạn mở rộng những gì đã build.
🧪 Lab B (Baseline)
  • TF-IDF keyword retrieval (sparse)
  • RandomForest với 120 records CSV
  • Template-based executive report
  • FLAN-T5 optional explainer
  • Session memory (Python list)
  • Gradio UI đơn giản
  • FastAPI stub
🏆 Capstone (Target)
  • FAISS + sentence-transformers (dense semantic)
  • Expanded dataset + SHAP explanations
  • OpenAI GPT-4o executive narrative
  • LangChain multi-step agent
  • SQLite persistent memory (production-grade)
  • Gradio UI nâng cao với SHAP charts
  • FastAPI với proper endpoints + auth
🧠
Tại sao các upgrade này quan trọng?
FAISS: Dense vector search tốt hơn TF-IDF khi query viết tắt, khác ngữ pháp nhưng cùng ý nghĩa (semantic similarity thay vì keyword match).
SHAP: Basel IV yêu cầu bank phải giải thích automated credit decisions — SHAP cho phép "explain" từng prediction cụ thể.
GPT-4o: Chất lượng narrative cao hơn FLAN-T5, hiểu context tốt hơn, phù hợp cho C-level executive reports.
LangChain: Orchestrate multi-step reasoning — agent có thể tự quyết định dùng tool nào, khi nào.
01
Phạm vi Công việc — Scope of Work
3 module chính, chia theo tuần thực hiện
🔍
Module A — Enhanced Knowledge Retrieval
Tuần 1 · Nâng cấp Layer 1 của pipeline
Technical
  • 1
    Thay TF-IDF bằng FAISS + Sentence-Transformers

    Cài faiss-cpu và sentence-transformers. Encode toàn bộ enterprise_knowledge.csv thành dense vectors với model all-MiniLM-L6-v2. Build FAISS index. So sánh recall score với TF-IDF baseline.

  • 2
    Mở rộng Knowledge Base

    Thêm ít nhất 20 issue patterns mới covering: crypto fraud, ESG reporting, real-time FX hedging, AML compliance, digital onboarding. Tổng tối thiểu 30 patterns.

  • 3
    Semantic Search API

    Implement POST /api/v1/search endpoint nhận query text, return top-3 kết quả với similarity score, department routing và confidence. Viết unit tests.

  • 4
    Cross-domain Demo

    Demo "pattern transfers across domains" (Slide 15): test cùng pipeline với queries Healthcare ("delayed patient billing reconciliation"), Legal ("contract review bottleneck"), HR ("payroll calculation errors"). Kết quả routing có đúng không?

🤖
Module B — Advanced ML + Explainability
Tuần 1–2 · Nâng cấp Layer 2–3 + SHAP (Basel IV)
Technical
  • 1
    Expanded Dataset (≥ 300 records)

    Mở rộng project_cost_data.csv lên 300+ records. Thêm 2 features mới: seniority_mix (junior/mid/senior ratio) và tech_complexity (1–5). Retrain RandomForest, đo R² cải thiện.

  • 2
    SHAP Explanations (Slide 14 — Basel IV)

    Cài shap. Với mỗi prediction, generate SHAP waterfall plot giải thích: feature nào đẩy cost lên, feature nào kéo xuống, bao nhiêu $. Đây là yêu cầu compliance cho automated credit decisions.

  • 3
    Model Comparison

    So sánh RandomForest vs GradientBoostingRegressor vs XGBoost. Dùng cross-validation 5-fold. Report bảng: R², MAE, inference time, explainability support.

  • 4
    Confidence Calibration

    Thay 95% CI đơn giản bằng proper prediction interval. Vẽ coverage plot kiểm tra xem CI có thực sự cover 95% test samples không.

✍️
Module C — LangChain Agent + GPT-4o + Production UI
Tuần 2–3 · Nâng cấp Layer 4–5 + Frontend + API
Capstone
  • 1
    LangChain Multi-step Agent (Slide 16)

    Build LangChain agent với 3 tools: SearchKB(query), PredictBudget(params), GenerateReport(context). Agent tự quyết định thứ tự gọi tools dựa trên user query phức tạp. Test với câu hỏi multi-hop: "So sánh budget nếu tăng team từ 10 lên 20 người với risk level khác nhau".

  • 2
    OpenAI GPT-4o Executive Narrator (Slide 6, 7)

    Thay FLAN-T5 bằng GPT-4o API (openai package). Implement Role Prompting + Structured Output pattern (Module 4). So sánh chất lượng narrative với FLAN-T5 baseline. Implement fallback: nếu không có API key → dùng template.

  • 3
    SQLite Persistent Memory (Slide 13)

    Thay Python list memory bằng SQLite database với asyncpg hoặc sqlite3. Schema: sessions(id, business, issue, budget, urgency, timestamp). Implement query history, search qua lịch sử, export CSV.

  • 4
    Gradio UI Pro + FastAPI Production

    Nâng cấp UI với: SHAP waterfall chart, multi-query conversation panel, session history table, export to PDF report. FastAPI: implement /analyze, /history, /export endpoints với proper error handling và basic auth.

Quy trình Thực hiện
3 tuần · 4 milestones · Sprint-based development
TUẦN 1 (Ngày 1–5) · Foundation Upgrade
🔍 FAISS + Expanded KB + Dataset
  • 📌 Ngày 1–2: Cài faiss-cpu, sentence-transformers. Encode KB thành vectors. Build FAISS index. So sánh TF-IDF vs FAISS recall trên 10 test queries.
  • 📌 Ngày 3: Mở rộng KB lên 30+ patterns (thêm các domains mới). Kiểm tra routing accuracy.
  • 📌 Ngày 4: Mở rộng dataset lên 300+ records. Thêm 2 features mới. Retrain RandomForest.
  • 📌 Ngày 5: Cài SHAP. Generate waterfall plot cho 3 test cases. Viết unit tests cho Module A.
  • 📦 Deliverable W1: core/database_v2.py (FAISS) · Updated dataset · SHAP notebook · Test results
TUẦN 2 (Ngày 6–10) · Intelligence Layer
🤖 LangChain + GPT-4o + SQLite
  • 📌 Ngày 6–7: Build LangChain agent với 3 tools. Test multi-hop query: "So sánh budget 3 scenarios khác nhau". Agent phải tự gọi PredictBudget 3 lần và tổng hợp kết quả.
  • 📌 Ngày 8: Integrate OpenAI GPT-4o API. Implement Role Prompting + fallback. So sánh output với FLAN-T5.
  • 📌 Ngày 9: Build SQLite memory layer. Schema + CRUD operations + query history API.
  • 📌 Ngày 10: Kết nối tất cả: FAISS → LangChain → ML+SHAP → SQLite → GPT-4o. Test full pipeline end-to-end. Fix bugs.
  • 📦 Deliverable W2: core/agent.py (LangChain) · core/memory_db.py (SQLite) · GPT-4o integration · E2E test passed
TUẦN 3 (Ngày 11–14) · Production & Polish
🚀 UI Pro + FastAPI + Testing
  • 📌 Ngày 11: Build Gradio UI Pro: sidebar navigation, SHAP chart tab, agent chat panel, status badges.
  • 📌 Ngày 12: FastAPI endpoints: /api/v1/analyze, /api/v1/history, /api/v1/export. Swagger docs auto-generated.
  • 📌 Ngày 13: Integration testing. Performance benchmark: measure latency của từng layer. Error handling và logging.
  • 📌 Ngày 14: Record demo video (5 phút). Viết báo cáo kỹ thuật. Chuẩn bị GitHub repo. Final review.
  • 📦 Deliverable W3: Production UI · FastAPI · Tests · Video demo · Technical report
TUẦN 4 (Ngày 15) · Demo Day
🏁 Presentation & Submission
  • 📌 Trình bày demo live: 10 phút demo + 5 phút Q&A
  • 📌 Nộp: GitHub repo link + demo video URL + báo cáo PDF
  • 📌 Peer review: mỗi nhóm review 1 nhóm khác theo rubric

🏆 Capstone Challenge — Bonus Track

Nếu nhóm muốn vượt ra ngoài spec, đây là những thách thức nâng cao:

  • Real dataset integration: Kết nối Kaggle "Give Me Some Credit" (150K records) thay synthetic CSV. Handle imbalanced data, feature engineering, model retraining.
  • Production deployment: Docker container + deploy lên Hugging Face Spaces hoặc Railway. CI/CD với GitHub Actions.
  • RAG Architecture: Implement Retrieval-Augmented Generation đúng chuẩn — LLM đọc context từ FAISS trước khi generate, không chỉ nhận KB result dưới dạng text.
  • Multi-tenant: Hỗ trợ nhiều công ty khác nhau với KB riêng biệt. Implement proper data isolation.
